## `make_datacite_context.py`

**What it was supposed to do:** Generate a JSON-LD context automatically from the DataCite XSD schema file.

**Problem:** The script is broken and will crash. Line 8 calls `schema.to_dict('metadata.xsd')` — it passes the XSD file path to itself as if it were an XML instance document. An XSD is a schema definition, not data. You cannot convert a schema into a dictionary this way.

**Fix needed:** Either (a) delete this script (the context is already manually maintained in `rdf-vocabulary-staging/context/fullcontext.jsonld`), or (b) rewrite it to use `xmlschema` correctly by loading the XSD as a schema and then converting an actual XML instance file.

## `datacite_api.jsonld`

**What it was supposed to be:** A version of the bundled validation profile (`integrated.json`) with its `$id` pointing to `schema.datacite.org` (production namespace).

**Problems:**
1. Uses `"additionalProperties": true` on the `identifiers` array items — this allows any extra fields to pass validation, defeating the purpose of strict validation.
2. The `examples` array inside `identifiers` has a structural error: the example value is a single object `{ "identifier": "...", "identifierType": "..." }` not wrapped in an array, so it does not match the declared `type: array` of the parent.
3. Contains duplicate `$comment` key at the top level (same issue as `integrated.json`).
4. The `$id` points to `schema.datacite.org/linked-data/linked-data/...` — note the doubled `linked-data/linked-data` path segment, which is wrong.

**Fix needed:** Decide whether this file is meant to be the production-namespace version of `integrated.json`. If yes, fix the four issues above and move it to `validation-and-conversion/schema-profiles/` alongside the other profiles.

## `main_context.jsonld`

**What it was supposed to be:** A JSON-LD context mapping all DataCite JSON fields to IRIs, intended as a standalone context file.

**Problems:**
1. The `datacite:` prefix resolves to `https://selgebali.github.io/datacite4.6-jsonld/main_context.jsonld#` — a personal GitHub Pages URL, not an official DataCite namespace.
2. Multiple `*Scheme` keys (e.g., `resourceTypeGeneralScheme`, `contributorTypeScheme`) reference external JSON-LD files like `https://selgebali.github.io/datacite4.6-jsonld/resourceTypeGeneral.jsonld` — these files **do not exist** in the repository. Any JSON-LD processor trying to resolve these will get a 404.
3. This file is a partial earlier draft that was superseded by `rdf-vocabulary-staging/context/fullcontext.jsonld` and `production-namespace/context/datacite.jsonld`. It adds no unique value.

**Fix needed:** Either delete this file (it is replaced by better versions) or, if keeping it, replace the personal GitHub URLs with the correct DataCite namespace IRIs and remove the broken `*Scheme` references.

## `XML-Shaped-JSON.json`

**What it is:** A JSON version of the DataCite example XML, produced using the `xml-js` library with XML attribute structure preserved (`@attrs`, `@value` patterns). This was used to test XML ↔ JSON roundtripping.

**Problem:** This is an intermediate working file from an experiment, not a canonical artifact. The structure (with `@attrs` keys) is not the same as the DataCite REST API JSON format. Using this directly would confuse anyone expecting standard DataCite JSON.

**Fix needed:** If the xml-js roundtrip approach is being documented, keep this file in a clearly labelled `examples/xml-roundtrip/` folder with a README explaining the structure. Otherwise delete it.
